# blog-draft

合同会社AdMoat（Meta広告運用代理店）のコーポレートサイトのブログ記事を、スプレッドシート駆動CMS（`blog.dc.html` + Google Apps Script + 「ブログ」シート）にそのまま投入できる形で下書きする。

## 手順

1. **既存記事の把握**：`blog.dc.html` を読み、`/* POSTS:START */`〜`/* POSTS:END */` の中身から、既存記事のタイトル・カテゴリを確認する。同じテーマ・切り口の重複は避ける。
2. **本数の決定**：ユーザーが本数を指定していなければ **3本** を作成する。
3. **テーマの決定**：
   - ユーザーがテーマを指定していれば、それに従う。
   - 指定がなければ、以下のカテゴリからバランスよく選び、まだ書かれていない切り口を考えて自動決定する：
     - 運用ノウハウ（Meta広告全般の実践的なコツ）
     - 店舗集客（実店舗・エリアビジネス向け）
     - 採用広告（Meta広告での採用母集団形成）
     - EC・サービス（EC/対人サービス業の獲得改善）
   - SEOを意識し、検索されそうな悩み・疑問ベースの具体的なタイトルにする（例:「〜の理由」「〜の3つのポイント」「〜な人向け」等、既存3記事のトーン・粒度に揃える）。
4. **本文のトーン**：既存記事と同じ、である調ではなく丁寧語・断定しすぎない解説調。1記事の本文は4〜6文程度を1〜2段落（改行区切り）でまとめる。概要（概要文）は1文で本文の要約。
5. **出力形式**：スプレッドシートの列にそのまま転記できるよう、記事ごとに以下のラベル付きブロックで出力する（表形式のTSVにはしない。本文に改行を含むため、セル単位でコピペしてもらう前提）。

```
【記事1】
タイトル: ...
カテゴリ: ...
日付: YYYY.MM.DD（今日の日付）
概要: ...
本文:
（1段落目）
（2段落目があれば）
画像URL: （空欄でOK）
スラッグ: （空欄でOK・自動生成）
公開: チェックを入れる
```

6. 出力の最後に一言、「「ブログ」シートに1行ずつ貼り付けて、公開列をチェックしてから同期を実行してください」と案内する。

## 注意

- このスキルはスプレッドシートやGitHubに直接書き込まない（下書きをチャットに出力するだけ）。同期の仕組み上、スプレッドシートに反映されていない記事は次回同期時に一覧から消えるため、必ずシートへの転記を経由させる。
- 日付は必ず実行時点の実際の日付を使う（架空の未来日付を作らない）。
